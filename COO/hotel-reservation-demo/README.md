# Système de réservation d'hôtels

Demo universitaire front-end d'un systeme de reservation d'hotels. Le projet presente les principaux ecrans et workflows issus de la conception UML: utilisateur, hotel, chambre, disponibilite, reservation, paiement, avis, notification et compte fidelite.

## Technologies utilisees

- HTML
- CSS
- JavaScript
- localStorage pour conserver certaines donnees de demo

## Comment lancer localement

Ouvrir directement le fichier `index.html` dans un navigateur moderne.

Aucun backend, aucune base de donnees, aucun framework et aucune installation Node.js ne sont necessaires.

## Deploiement GitHub Pages

1. Ajouter le dossier `hotel-reservation-demo` dans un depot GitHub.
2. Pousser les fichiers `index.html`, `style.css`, `script.js` et `README.md`.
3. Dans GitHub, ouvrir `Settings > Pages`.
4. Choisir la branche a publier.
5. Si le dossier est a la racine, choisir `/root`. Sinon, placer les fichiers de l'application a la racine du depot ou dans le dossier publie.
6. Ouvrir l'URL GitHub Pages generee.

## Note importante

Ce projet est un MVP/demo universitaire front-end uniquement. Il ne contient pas de vraie authentification, pas de vrai paiement, pas de backend, pas d'API et pas de base de donnees. Les donnees sont creees directement dans `script.js` pour montrer visuellement le fonctionnement du systeme pendant une presentation.
